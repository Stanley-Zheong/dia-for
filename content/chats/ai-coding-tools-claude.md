## User

我想写一个公开站，比较不同 AI 编程工具的能力。应该怎么组织？

## Claude

我会把文章组织成“场景优先”的结构：

- 从一个真实项目出发。
- 记录每个工具如何理解代码。
- 比较它们提出方案时的取舍。
- 最后看它们是否能完成可验证的改动。

这样读者看到的是工作流差异，而不是营销词汇。

## User

那不同模型观点的提取要怎么做？

## Claude

先保留原始对话，再在旁边生成派生层：

1. 共识。
2. 分歧。
3. 独特建议。
4. 可以引用的原文依据。

不要让总结覆盖原文。
